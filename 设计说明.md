### Auth API
| 方法 | 路径 | 功能 |
| --- | --- | --- |
| POST | `/api/auth/register` | 注册 -> 创建用户 + 签发 JWT + 设置 httpOnly Cookie |
| POST | `/api/auth/login` | 登录 -> 验证密码 + 签发 JWT + 设置 httpOnly Cookie |
| GET | `/api/auth/me` | 获取当前用户（Cookie / Bearer Token） |
| PATCH | `/api/auth/me` | 修改个人信息（姓名）/ 修改密码（需验证当前密码）+ 重签 JWT |

### 认证流程
1. **注册**：填写昵称/邮箱/密码 -> `POST /api/auth/register` -> 哈希密码 + 创建用户 + 签发 JWT -> 设置 httpOnly Cookie + localStorage -> 跳转仪表盘
2. **登录**：填写邮箱/密码 -> `POST /api/auth/login` -> 验证密码 + 签发 JWT -> 跳转仪表盘
3. **会话维持**：AppShell 加载时 `GET /api/auth/me`（自动携带 Cookie）-> 获取用户信息 -> 未登录则跳转 `/login`
4. **退出**：清除 localStorage + Cookie -> 跳转 `/login`
5. **修改资料**：设置页「个人信息」标签从 `GET /api/auth/me` 拉取当前用户 -> 编辑姓名/修改密码 -> `PATCH /api/auth/me` -> 重签 JWT + 更新 Cookie

### 端到端验证（真实跑通）
- 4 个演示账号登录（Owner/Admin/Editor/Viewer）均返回 JWT + 用户信息 ✅
- 错误密码返回 401「邮箱或密码不正确」✅
- 新用户注册 -> 重复邮箱返回 409「该邮箱已被注册」✅
- `/api/auth/me` 携带 Token 返回用户、不携带返回 null ✅
- AppShell 动态显示当前用户名 + 角色徽章 + 退出登录菜单 ✅
- 设置页「个人信息」从 `/api/auth/me` 动态加载当前用户（姓名/邮箱/角色/套餐），不再硬编码 ✅
- 修改姓名 `PATCH /api/auth/me` -> 重签 JWT + 持久化 ✅
- 修改密码需验证当前密码，旧密码失效、新密码可登录 ✅

> 🔌 生产环境：替换 SHA-256 为 PBKDF2/bcrypt（`session.ts` 已实现）；接入 NextAuth.js 支持 OAuth（Google/GitHub）；中间件层校验 JWT 保护路由。

## 七、智能问答后端 / RAG 引擎（第 5–6 周）

### RAG 引擎 `src/lib/rag/`（全异步化，支持 LLM Provider）
- `embeddings.ts`：本地哈希嵌入（拉丁词 + CJK 字符/二元组 → 2048 维 L2 归一化），演示模式回退
- `chunker.ts`：字符级带重叠切片
- `vector-store.ts`：异步 `indexChunks()`（批量嵌入）+ 内存向量索引（余弦相似，支持任意维度）
- `retriever.ts`：异步 `retrieve()` → `embedText()` 查询嵌入 → Top-K 检索
- `generator.ts`：`generateStream()`（LLM 流式生成 + 引用解析）/ `generateAsync()` / `generate()`（抽取式回退）
- `indexer.ts`：异步 `indexDocument()` → 文档就绪后自动切片 + 嵌入 + 入库
- **LLM Provider**（`src/lib/llm/provider.ts`）：配置 `OPENAI_API_KEY` 时使用 OpenAI 嵌入（1536 维）+ 流式生成；未配置时回退本地哈希嵌入 + 抽取式生成

> 文档就绪即自动索引；查询时异步检索 + LLM/抽取式生成 + SSE 流式输出 + 引用，全部基于真实文档内容。

## 十五、生产化接入（Provider 适配层）

所有核心模块均通过 Provider 适配层实现「配置即切换」——设置环境变量即启用真实服务，未配置则自动回退到本地演示模式，业务代码无需改动。

### LLM Provider `src/lib/llm/`
- `provider.ts`：统一接口 `embedText()` / `chatComplete()` / `chatStream()`
- **OpenAI 兼容**：支持 OpenAI / Azure / vLLM / Ollama / DeepSeek / Moonshot / 硅基流动
- **配置解析优先级**：① 用户在设置页配置的已启用模型（运行时）-> ② 环境变量（部署时）-> ③ 本地演示模式
- 嵌入：远程 `text-embedding-3-small`（1536 维）/ 本地哈希嵌入（2048 维）
- 对话：远程 `gpt-4o-mini` 流式生成 / 本地抽取式生成
- RAG 引擎已全异步化：`retrieve` / `generateStream` / `indexChunks` 均支持 LLM 嵌入与流式生成
- `isLLMEnabled()` / `llmLabel()` 等改为 async，异步解析用户模型配置

> 环境变量配置：`OPENAI_API_KEY` + `CHAT_MODEL` + `EMBEDDING_MODEL` + `OPENAI_BASE_URL`
> 运行时配置：设置页「AI 模型」标签，无需重启即可切换模型（见下方「外部模型管理」）

### Payment Provider `src/lib/billing/provider.ts`
- `createCheckoutSession()`：Stripe Checkout Session（订阅/一次性支付）→ 返回跳转 URL
- `verifyWebhook()`：Stripe Webhook 签名验证
- `/api/billing/webhook`：Webhook 端点 → 自动确认支付 → 升级订阅
- 未配置时回退到模拟支付（2 步流程：创建订单 → 支付）

> 配置：`STRIPE_SECRET_KEY` + `STRIPE_WEBHOOK_SECRET` + `STRIPE_PRICE_PRO` + `STRIPE_PRICE_ENT`

### Database `src/lib/db/`
- `schema.prisma`：完整 PostgreSQL schema（User / Team / KB / Document / Conversation / Message / AgentTask / Subscription / Invoice / ApiKey / Session / LoginEvent）
- `client.ts`：Prisma 懒加载单例（`getDb()` → 检测 `@prisma/client` 是否安装）
- `repository.ts`：Repository 模式示例——`isDbEnabled()` 时查 Prisma，否则用内存 store
- 所有 `store.ts` 均基于 `globalThis` 内存存储，接口与 Prisma 对齐，可平滑替换

> 配置：`DATABASE_URL` + `pnpm add @prisma/client && npx prisma migrate deploy`

### Storage `src/lib/storage/`
- `saveFile()` / `readFile()` / `deleteFile()`：统一文件操作接口
- S3 兼容：AWS S3 / MinIO / Cloudflare R2（通过 `S3_ENDPOINT`）
- 未配置时回退到本地 `.uploads/` 目录

> 配置：`S3_ENDPOINT` + `S3_BUCKET` + `S3_ACCESS_KEY` + `S3_SECRET_KEY`

### Rate Limiting `src/middleware.ts`
- 滑动窗口限流：每 IP 每分钟 N 次（默认 60，`RATE_LIMIT_PER_MIN` 可配）
- 应用于 `/api/*`（排除 SSE 流式端点 `/api/chat` `/api/agent/run`）
- 返回 `429` + `Retry-After` + `X-RateLimit-*` 响应头
- 🔌 生产环境替换为 Redis/Upstash 实现多实例限流

### Auth `src/lib/auth/session.ts`
- JWT 签发/验证（Web Crypto API HMAC-SHA256，零依赖）
- PBKDF2 密码哈希（10 万次迭代）
- `createToken()` / `verifyToken()` / `getUserFromRequest()` / `hashPassword()` / `verifyPassword()`
- 🔌 生产环境替换为 NextAuth.js / better-auth（OAuth + CSRF + 会话管理）

### 外部模型管理 `src/lib/models/`
用户可在设置页「AI 模型」标签导入外部 LLM，无需重启即可驱动 RAG 与 Agent。

- `types.ts`：`ProviderId`（6 个预设）+ `ModelConfig` / `ModelConfigSafe`（API Key 脱敏）
- `store.ts`：模型 CRUD + 6 个提供商预设（OpenAI / DeepSeek / Moonshot / 硅基流动 / Ollama / 自定义）+ `getActiveModel()`（默认 > 首个已启用）+ `sanitize()`（Key 脱敏 `sk-x••••xxxx`）

| 方法 | 路径 | 功能 |
| --- | --- | --- |
| GET | `/api/models` | 模型列表（脱敏）+ 提供商预设 |
| POST | `/api/models` | 添加模型（校验 Key / Base URL / 模型名） |
| PATCH | `/api/models/[id]` | 更新（启停 / 设为默认 / 改配置） |
| DELETE | `/api/models/[id]` | 删除模型 |
| POST | `/api/models/test` | 连接测试（发 1 条 `Hi` 请求，15s 超时） |

- 设为默认时自动启用，且全局仅一个默认模型
- 连接测试结果（ok/失败 + 延迟）持久化到模型配置，UI 展示
- 添加表单选择提供商后自动填充 Base URL + 模型名候选（datalist）
- 🔌 生产环境：API Key 加密存储（KMS / Vault）；模型配置落 Prisma

### Provider 状态可视化
管理后台「系统配置」卡新增**生产环境接入状态**面板，实时展示 6 项 Provider 的启用情况（LLM / 嵌入 / 数据库 / 存储 / 支付 / 限流），绿点=已启用、灰点=演示模式。

### 端到端验证（演示模式）
- 限流：连续请求 54 次后返回 `429 Too Many Requests` + `Retry-After` ✅
- Provider 状态：`/api/admin/config` 返回 6 项 Provider 状态（全部演示模式）✅
- Chat SSE：异步 RAG 检索 + 流式生成（演示模式抽取式）正常输出 ✅
- Agent SSE：异步检索 + 4 Agent 编排正常执行 ✅
- Checkout：模拟模式返回 2 步流程订单 ✅
